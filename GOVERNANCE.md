# Repository Governance – Public Summary

Worshipping Sekhmet maintains a structured, multi-level repository architecture to ensure that information is handled according to its classification level and the Need-to-Know principle.

This public summary provides an overview of the governance principles. The full governance document is classified as **WS RESTRICTED** and is **not available in this repository**. It is maintained in a controlled environment.

## Key Principles

- **Protection Levels**: Repositories are organised according to classification (Public, WS RESTRICTED, and higher compartments).
- **Gateway Control**: Information flow between protection levels is defined and controlled by the compartment with the higher protection requirement.
- **Approval Process**: Content intended for broader release requires appropriate oversight and approval, including Meaningful Human Oversight (MHO) where required.
- **Traceability**: Released versions of templates and key documents are archived and tagged for long-term accountability.
